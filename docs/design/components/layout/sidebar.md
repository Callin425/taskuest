# Sidebar コンポーネント設計

## 1. 概要

`Sidebar` は、Taskuest v1 における認証後画面の共通サイドナビゲーションコンポーネントである。  
主に以下の用途で使用する。

- 画面遷移メニューの表示
- 現在画面のナビゲーション補助
- サイドメニューの開閉（展開 / 縮小）

本コンポーネントは、**ナビゲーションUIと表示状態の制御に責務を限定**し、画面遷移ロジックや業務状態は持たせない。

---

## 2. 目的

- アプリ全体のナビゲーションUIを統一する
- 各画面への導線を明確にする
- 展開 / 縮小による表示切り替えを提供する
- 過剰な機能追加を避け、v1 時点で保守しやすい設計にする

---

## 3. 配置場所

```txt
src/components/layout/Sidebar/
├─ Sidebar.tsx
├─ Sidebar.test.tsx
└─ index.ts
```

---

## 4. 責務

`Sidebar` が担当する責務は以下とする。

- サイドナビゲーション領域の表示
- メニュー項目の一覧表示
- 展開 / 縮小状態の表示切り替え
- メニュークリックのイベント受け付け
- 現在選択中メニューの表示
- 共通スタイルの適用

---

## 5. 担当しない責務

以下は `Sidebar` の責務に含めない。

- 画面遷移処理（router 操作など）
- 業務ロジックの管理
- 権限による表示制御
- メニュー構造の動的生成
- 複雑なアニメーション制御
- API通信

これらは画面側または上位コンポーネントで管理する。

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| items | `SidebarItem[]` | ○ | メニュー一覧 |
| activeKey | `string` | △ | 現在選択中のメニュー |
| isCollapsed | `boolean` | ○ | 縮小状態かどうか |
| onToggle | `() => void` | △ | 展開 / 縮小切り替えイベント |
| onSelect | `(key: string) => void` | ○ | メニュー選択時イベント |
| className | `string` | △ | 追加調整が必要な場合の拡張用クラス |

### SidebarItem 型

```ts
type SidebarItem = {
  key: string
  label: string
  icon?: React.ReactNode
}
```

### 7.1 設計判断

- `items` は外部から渡し、メニュー内容の変更に対応しやすくする
- `activeKey` により選択状態を制御する
- `isCollapsed` により展開 / 縮小表示を制御する
- 遷移処理は `onSelect` に委譲する
- `icon` は縮小時のナビゲーションに必要な要素として扱う
- `onToggle` が未指定の場合は、トグルボタンを表示しない、または押下不可とする

---

## 8. 表示仕様

### 8.1 通常（展開）状態
- アイコン + ラベルを表示する

### 8.2 縮小状態
- アイコンのみ表示する
- ラベルは非表示にする

### 8.3 メニュー選択
- `activeKey` に一致する項目を強調表示する

### 8.4 メニュークリック
- 項目クリック時に `onSelect(key)` を呼び出す

### 8.5 トグル操作
- トグルボタン押下時に `onToggle` を呼び出す
- `onToggle` が未指定の場合は、トグル操作を提供しない

---

## 9. アクセシビリティ

- `nav` 要素を使用し、ナビゲーション領域であることを明示する
- メニュー項目は `button` 要素を使用してクリック可能であることを明確にする
- キーボード操作で各項目にフォーカス可能にする
- アイコンのみ表示時でも意味が分かるラベルを付与する
- 現在選択中の項目が支援技術にも伝わるよう配慮する

---

## 10. 想定利用例

### 10.1 認証後共通レイアウト

以下のメニューを `items` として渡す。

- タスク一覧
- プロジェクト一覧
- ユーザー設定

### 10.2 task-list 画面

- `activeKey` に `task-list` を指定する
- タスク一覧メニューを強調表示する
- メニュー選択時は上位コンポーネントで画面遷移する

### 10.3 縮小表示時

- アイコンのみを表示する
- ラベルは視覚的に非表示にする
- アイコンだけでは意味が伝わりにくい場合は、代替ラベルを用意する

## 11. 再定義コンポーネント候補

- `AppSidebar`

※ v1 初期段階では、共通 `Sidebar` をそのまま利用する。

---

## 12. 実装方針

- `Sidebar.tsx` を単体で実装する
- 認証後レイアウト内で利用する
- 状態管理は上位コンポーネントで行う
- UIは最小構成で開始する
- メニュー構造が複雑化した場合のみ、再定義コンポーネントや設定ファイル化を検討する

---

## 13. テスト観点

### 13.1 単体テスト対象
- メニュー一覧が表示されること
- `activeKey` が反映されること
- `isCollapsed` による表示切り替えが行われること
- `onSelect` が呼ばれること
- `onToggle` が指定されている場合に呼ばれること

### 13.2 テストしないこと
- 画面遷移処理
- 業務ロジック
- 権限による表示制御
- API通信結果による表示制御
