# Toast コンポーネント設計

## 1. 概要

`Toast` は、Taskuest v1 における一時的な通知表示用の共通コンポーネントである。  
主に以下の用途で使用する。

- タスク作成成功通知
- タスク更新成功通知
- タスクアーカイブ成功通知
- エラー発生時の通知
- 情報メッセージの通知

本コンポーネントは、**短時間表示される通知UIの表示責務に限定**し、通知の発火条件や業務ロジックそのものは持たせない。

---

## 2. 目的

- 通知UIの見た目と振る舞いを統一する
- 成功 / エラー / 情報通知を共通化する
- task-list を含む複数画面で再利用できる通知基盤を作る
- 過剰な状態管理や通知管理機構を避け、v1 時点で保守しやすい設計にする

---

## 3. 配置場所

```txt
src/components/ui/Toast/

├─ Toast.test.tsx
└─ index.ts
```

---

## 4. 責務

`Toast` が担当する責務は以下とする。

- 通知メッセージの表示
- 通知種別に応じた見た目の切り替え
- 閉じる操作の受け付け
- 表示 / 非表示状態に応じた描画
- 共通スタイルの適用

---

## 5. 担当しない責務

以下は `Toast` の責務に含めない。

- 通知を出す業務条件の判定
- グローバルな通知キュー管理
- 複数通知の並列制御
- 画面全体の状態管理
- API通信結果の判定
- 高度なアニメーション制御
- 通知履歴の保存

これらが必要になった場合は、画面側または別レイヤーで対応する。

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| message | `string` | ○ | 通知メッセージ |
| type | `'success' \| 'error' \| 'info'` | △ | 通知種別。デフォルトは `info` |
| isOpen | `boolean` | ○ | 表示状態 |
| onClose | `() => void` | △ | 閉じる操作時のイベント |
| showCloseButton | `boolean` | △ | 閉じるボタンを表示するかどうか。デフォルトは `true` |
| className | `string` | △ | 追加調整が必要な場合の拡張用クラス |

### 7.1 設計判断

- `type` は v1 で必要になる最低限の種別に絞る
- `variant` ではなく `type` とする
- `title` は現時点では持たせない
- 自動非表示時間はコンポーネント内で持たせない
- `isOpen` によって表示 / 非表示を制御する
- `showCloseButton=true` の場合は、原則として `onClose` を指定する
- `onClose` が未指定の場合は、閉じるボタンを表示しない、または押下不可とする

---

## 8. 表示仕様

### 8.1 通常時
- `isOpen` が `true` の場合に通知を表示する
- 通知メッセージを表示する

### 8.2 種別ごとの表示
- `success` の場合、成功通知用の見た目を適用する
- `error` の場合、エラー通知用の見た目を適用する
- `info` の場合、情報通知用の見た目を適用する

### 8.3 閉じる操作
- `showCloseButton=true` かつ `onClose` が指定されている場合、閉じるボタンを表示する
- 閉じるボタン押下時は `onClose` を呼び出す
- `onClose` が未指定の場合は、閉じる操作を提供しない

### 8.4 非表示時
- `isOpen` が `false` の場合は描画しない、または非表示状態とする

---

## 9. アクセシビリティ

- 通知内容が認識しやすいテキストであること
- 必要に応じて `role="status"` または `role="alert"` を付与する
- 閉じるボタンには意味の分かるラベルを付与する
- キーボード操作で閉じるボタンに到達できること

## 12. 実装方針

- `Toast.tsx` を単体で実装する
- task-list の作成 / 編集 / 更新完了時に使用し、実利用を通じて不足を確認する
- 必要になるまで通知管理機構を広げすぎない
- 通知の発火条件や表示時間制御は画面側で管理する

---

## 13. テスト観点

### 13.1 単体テスト対象
- `isOpen=true` のときにメッセージが表示されること
- `isOpen=false` のときに非表示であること
- `type` に応じた表示が反映されること
- `showCloseButton=true` かつ `onClose` 指定時に閉じるボタンが表示されること
- 閉じるボタン押下時に `onClose` が呼ばれること
- `onClose` 未指定時に閉じる操作を提供しないこと

### 13.2 テストしないこと
- 業務ロジックに基づく通知表示判定
- API通信成否の判定
- 複数通知の管理
- 自動クローズのタイマー制御
