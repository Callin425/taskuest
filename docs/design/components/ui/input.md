# Input コンポーネント設計

## 1. 概要

`Input` は、Taskuest v1 における単一行テキスト入力用の共通コンポーネントである。  
主に以下の用途で使用する。

- タスク名入力
- 検索キーワード入力
- ユーザー設定画面における単一行テキスト入力

本コンポーネントは、**単一行入力に責務を限定**し、複雑な入力種別や画面固有の振る舞いは持たせない。  
用途ごとの差分は、必要に応じて画面側または薄いラッパーコンポーネントで吸収する。

---

## 2. 目的

- 単一行入力UIの見た目と振る舞いを統一する
- ラベル、必須表示、エラー表示などの基本機能を共通化する
- task-list を含む複数画面で再利用できる土台を作る
- 過剰な汎用化を避け、v1 時点で保守しやすい設計にする

---

---

## 4. 責務

`Input` が担当する責務は以下とする。

- 単一行入力欄の表示
- ラベル表示
- 必須表示
- エラー文言表示
- disabled 状態の表示
- 共通スタイルの適用
- `input` 要素への基本的な props の受け渡し

---

## 5. 担当しない責務

以下は `Input` の責務に含めない。

- 複数行入力（`textarea`）
- プルダウン選択（`select`）
- 日付入力、数値入力など各入力種別専用UIの作り込み
- 入力値の業務バリデーション
- 検索実行処理
- 入力値の整形や変換
- アイコン表示前提の専用UI
- パスワード表示切替など複雑な付加機能

これらが必要になった場合は、別コンポーネントとして分離する。

## 6. 使用するHTML要素

- `label`
- `input`
- `p`（エラーメッセージ表示用）

---

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| id | `string` | ○ | `label` と関連付けるための識別子 |
| name | `string` | ○ | フォーム項目名 |
| label | `string` | △ | 項目ラベル。不要な場合は省略可 |
| type | `'text' \| 'email' \| 'password' \| 'search'` | △ | 入力タイプ。デフォルトは `text` |
| value | `string` | △ | 制御コンポーネントとして使う場合の値 |
| defaultValue | `string` | △ | 非制御コンポーネントとして使う場合の初期値 |
| placeholder | `string` | △ | プレースホルダー |
| disabled | `boolean` | △ | 非活性状態。デフォルトは `false` |
| required | `boolean` | △ | 必須項目かどうか。デフォルトは `false` |
| errorMessage | `string` | △ | エラーメッセージ |
| maxLength | `number` | △ | 最大文字数 |
| onChange | `(e: React.ChangeEvent<HTMLInputElement>) => void` | △ | 入力変更時のイベント |
| onBlur | `(e: React.FocusEvent<HTMLInputElement>) => void` | △ | フォーカスアウト時のイベント |
| className | `string` | △ | 画面側で追加調整が必要な場合の拡張用クラス |

### 7.1 設計判断

- `type` は v1 で必要になりうる最低限に絞る
- `variant` は現時点では持たせない  
- `prefix` / `suffix` / `icon` などは現時点では持たせない  
- `value` と `defaultValue` は両対応とするが、実際の使用時はどちらか一方に統一する  
- `errorMessage` は表示責務のみとし、判定ロジックは持たない  

---

## 8. 表示仕様

### 8.1 通常時
- ラベルを表示する
- 入力欄を表示する
- プレースホルダーが指定されている場合は表示する

### 8.2 必須時
- `required={true}` の場合、ラベル付近に必須表示を行う

### 8.3 エラー時
- `errorMessage` が存在する場合、入力欄をエラー状態の見た目に変更する
- 入力欄の下にエラーメッセージを表示する

### 8.4 disabled時
- 入力不可にする
- 視覚的にも非活性であることが分かるスタイルにする

---

## 9. アクセシビリティ

- `label` 要素と `input` 要素を `htmlFor` / `id` で関連付ける
- エラー表示時は `aria-invalid` を付与する
- 必要に応じて `aria-describedby` を利用する
- キーボード操作で自然にフォーカス移動できること
- プレースホルダーだけに依存せず、ラベルを基本とする

## 12. 実装方針

- `Input.tsx` を単体で実装する
- task-list で使用しながら不足を確認する
- 必要になるまで props や variant を増やさない
- 画面固有の見た目差分はラッパー側で吸収する

---

## 13. テスト観点

### 13.1 単体テスト対象
- ラベルが表示されること
- `placeholder` が表示されること
- `disabled` が反映されること
- `required` が反映されること
- `errorMessage` が表示されること
- `onChange` が呼ばれること
- `type` 指定が反映されること

### 13.2 テストしないこと
- 業務バリデーション
- 検索処理
- フォーム全体の送信挙動
