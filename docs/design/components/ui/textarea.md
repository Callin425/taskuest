# Textarea コンポーネント設計

## 1. 概要

`Textarea` は、Taskuest v1 における複数行テキスト入力用の共通コンポーネントである。  
主に以下の用途で使用する。

- タスク説明入力
- ユーザー設定画面における自由記述欄
- 将来的なメモ入力欄

本コンポーネントは、**複数行テキスト入力に責務を限定**し、単一行入力や画面固有の複雑な振る舞いは持たせない。  
用途ごとの差分は、必要に応じて画面側または薄いラッパーコンポーネントで吸収する。

---

## 2. 目的

- 複数行入力UIの見た目と振る舞いを統一する
- ラベル、必須表示、エラー表示などの基本機能を共通化する
- タスク説明入力を含む複数画面で再利用できる土台を作る
- 過剰な汎用化を避け、v1 時点で保守しやすい設計にする

---

## 3. 配置場所

```txt
src/components/ui/Textarea/
├─ Textarea.tsx
├─ Textarea.test.tsx
└─ index.ts
```

## 4. 責務

`Textarea` が担当する責務は以下とする。

- 複数行入力欄の表示
- ラベル表示
- 必須表示
- エラー文言表示
- disabled 状態の表示
- 共通スタイルの適用
- `textarea` 要素への基本的な props の受け渡し

---

## 5. 担当しない責務

以下は `Textarea` の責務に含めない。

- 単一行入力（`input`）
- プルダウン選択（`select`）
- 自動リサイズ
- 文字数カウンター表示
- Markdown入力補助
- リッチテキスト編集
- 入力値の業務バリデーション
- 入力値の整形や変換
- 画面固有の説明文や補助UIの作り込み

これらが必要になった場合は、画面側で対応するか、別コンポーネントとして分離する。

---

## 6. 使用するHTML要素

- `label`
- `textarea`
- `p`（エラーメッセージ表示用）

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| id | `string` | ○ | `label` と関連付けるための識別子 |
| name | `string` | ○ | フォーム項目名 |
| label | `string` | △ | 項目ラベル。不要な場合は省略可 |
| value | `string` | △ | 制御コンポーネントとして使う場合の値 |
| defaultValue | `string` | △ | 非制御コンポーネントとして使う場合の初期値 |
| placeholder | `string` | △ | プレースホルダー |
| disabled | `boolean` | △ | 非活性状態。デフォルトは `false` |
| required | `boolean` | △ | 必須項目かどうか。デフォルトは `false` |
| errorMessage | `string` | △ | エラーメッセージ |
| maxLength | `number` | △ | 最大文字数 |
| rows | `number` | △ | 表示行数。デフォルト値を持たせる |
| onChange | `(e: React.ChangeEvent<HTMLTextAreaElement>) => void` | △ | 入力変更時のイベント |
| onBlur | `(e: React.FocusEvent<HTMLTextAreaElement>) => void` | △ | フォーカスアウト時のイベント |
| className | `string` | △ | 画面側で追加調整が必要な場合の拡張用クラス |

### 7.1 設計判断

- `rows` は `Textarea` 固有の基本機能として持たせる
- `variant` は現時点では持たせない  
- `autoResize` は現時点では持たせない  
- `value` と `defaultValue` は両対応とするが、実際の使用時はどちらか一方に統一する  
- `errorMessage` は表示責務のみとし、判定ロジックは持たない  

---

## 8. 表示仕様

### 8.1 通常時
- ラベルを表示する
- 複数行入力欄を表示する
- プレースホルダーが指定されている場合は表示する

### 8.2 必須時
- `required={true}` の場合、ラベル付近に必須表示を行う

### 8.3 エラー時
- `errorMessage` が存在する場合、入力欄をエラー状態の見た目に変更する
- 入力欄の下にエラーメッセージを表示する

### 8.4 disabled時
- 入力不可にする
- 視覚的にも非活性であることが分かるスタイルにする

---

## 9. アクセシビリティ

- `label` 要素と `textarea` 要素を `htmlFor` / `id` で関連付ける
- エラー表示時は `aria-invalid` を付与する
- 必要に応じて `aria-describedby` を利用する
- キーボード操作で自然にフォーカス移動できること
- プレースホルダーだけに依存せず、ラベルを基本とする

## 12. 実装方針

- `Textarea.tsx` を単体で実装する
- タスク作成/編集UIで使用しながら不足を確認する
- 必要になるまで props や variant を増やさない
- 画面固有の見た目差分はラッパー側で吸収する

---

## 13. テスト観点

### 13.1 単体テスト対象
- ラベルが表示されること
- `placeholder` が表示されること
- `disabled` が反映されること
- `required` が反映されること
- `errorMessage` が表示されること
- `onChange` が呼ばれること
- `rows` 指定が反映されること

### 13.2 テストしないこと
- 業務バリデーション
- 入力文字数制限の業務ルール判定
- フォーム全体の送信挙動
