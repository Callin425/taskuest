# Button コンポーネント設計

## 1. 概要

`Button` は、Taskuest v1 におけるクリック操作用の共通 UI コンポーネントである。  
主に以下の用途で使用する。

- タスク作成ボタン
- タスク編集ボタン
- タスク完了ボタン
- タスクアーカイブボタン
- モーダル内の保存 / キャンセル操作

本コンポーネントは、**ボタンの見た目・サイズ・状態・クリックイベントの受け口に責務を限定**し、業務状態の判定や画面固有の処理は持たせない。

---

## 2. 目的

- アプリ全体でボタンUIの見た目と振る舞いを統一する
- 主操作 / 補助操作 / 破壊的操作の表現を共通化する
- テキストボタンとアイコンボタンの表示ルールを整理する
- 過剰な業務依存を避け、v1 時点で保守しやすい設計にする

---

## 3. 配置場所

```txt
src/components/ui/Button/
├─ Button.tsx
├─ Button.test.tsx
└─ index.ts
```

---

## 4. 責務

`Button` が担当する責務は以下とする。

- ボタン要素の表示
- variant に応じた標準スタイルの適用
- size に応じた表示調整
- disabled 状態の表示と制御
- クリックイベントの受け付け
- テキスト / アイコン / アイコンのみ表示の制御
- 共通スタイルの適用

---

## 5. 担当しない責務

以下は `Button` の責務に含めない。

- タスク完了状態などの業務判定
- 業務ルールに基づく表示文言の決定
- 特定画面固有の状態管理
- API通信
- 画面遷移処理
- 確認ダイアログの表示制御

これらは呼び出し元、画面側、または業務専用コンポーネントで対応する。

---

## 6. 使用するHTML要素

- `button`
- `span`（テキスト / アイコン表示用）

---

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| children | `React.ReactNode` | △ | ボタン内に表示するテキストや要素 |
| icon | `React.ReactNode` | △ | ボタン内に表示するアイコン |
| iconOnly | `boolean` | △ | アイコンのみで表示する場合に指定する。デフォルトは `false` |
| variant | `'primary' \| 'secondary' \| 'danger'` | △ | ボタンの役割に応じた見た目。デフォルトは `primary` |
| size | `'sm' \| 'md'` | △ | ボタンの表示サイズ。デフォルトは `md` |
| disabled | `boolean` | △ | 非活性状態。デフォルトは `false` |
| onClick | `() => void` | △ | クリック時のイベント |
| className | `string` | △ | 追加調整が必要な場合の拡張用クラス |
| type | `'button' \| 'submit' \| 'reset'` | △ | ボタン種別。デフォルトは `button` |

### 7.1 設計判断

- `variant` は業務種別ではなく、UI上の役割として定義する
- `size` は v1 で必要になる最低限の `sm` / `md` に絞る
- `iconOnly` は `variant` や `size` とは独立した表示形式として扱う
- 業務状態に応じた文言や見た目の判定は呼び出し元で行う
- 個別の軽微な調整が必要な場合のみ `className` を使用する

---

## 8. 表示仕様

### 8.1 通常時
- `children` に渡された内容を表示する
- `variant` と `size` に応じた見た目を適用する

### 8.2 variant ごとの表示

| variant | 用途 |
|---|---|
| primary | 画面内の主操作 |
| secondary | 補助操作 |
| danger | 削除・アーカイブなどの破壊的または注意が必要な操作 |

### 8.3 size ごとの表示

| size | 用途 |
|---|---|
| sm | カード内、一覧内などの小型ボタン |
| md | 通常サイズのボタン |

### 8.4 アイコン表示
- `icon` が指定され、`iconOnly=false` の場合はアイコン + テキストを表示する
- `iconOnly=true` の場合はアイコンのみを表示する
- `iconOnly=true` の場合、視覚的には正方形に近い形状とする

### 8.5 disabled 状態
- `disabled=true` の場合は押下不可にする
- 視覚的にも非活性であることが分かるスタイルにする

---

## 9. アクセシビリティ

- ボタン操作には `button` 要素を使用する
- `iconOnly=true` の場合は、実装時に `aria-label` などで意味を補足する
- disabled 状態が視覚的にも操作上も分かるようにする
- キーボード操作で自然にフォーカスできること
- 色だけに依存せず、文言やアイコンでも役割が伝わる構成にする

---

## 10. 想定利用例

### 10.1 タスク作成ボタン
- `variant="primary"` を指定する
- 画面内の主操作として表示する

### 10.2 タスク編集ボタン
- `variant="secondary"` を指定する
- カード内の補助操作として表示する

### 10.3 タスクアーカイブボタン
- `variant="danger"` を指定する
- 必要に応じて確認ダイアログと組み合わせる

### 10.4 アイコンのみボタン
- `iconOnly=true` を指定する
- 一覧内の省スペース操作に利用する

---

## 12. 実装方針

- `Button.tsx` を単体で実装する
- task-list の作成 / 編集 / 完了 / アーカイブ操作で使用しながら不足を確認する
- 必要になるまで variant や size を増やさない
- 同一の個別調整が複数箇所で発生する場合は、共通化または variant 見直しを検討する

---

## 13. テスト観点

### 13.1 単体テスト対象
- `children` が表示されること
- `variant` に応じた表示が反映されること
- `size` に応じた表示が反映されること
- `disabled` が反映されること
- `onClick` が呼ばれること
- `type` 指定が反映されること
- `icon` が表示されること
- `iconOnly=true` の場合にアイコンのみ表示になること

### 13.2 テストしないこと
- タスク完了などの業務ロジック
- API通信結果による表示制御
- 確認ダイアログの表示制御
