# Button コンポーネント定義

## 1. 概要
Button は、画面上のアクション操作に使用する共通 UI コンポーネントである。  
アプリケーション全体で操作表現の一貫性を保つことを目的とする。

## 2. 責務
本コンポーネントは、以下を責務とする。

- ボタン要素の描画
- 役割に応じた標準スタイルの適用
- サイズに応じた表示調整
- 非活性状態の制御
- クリックイベントの受け口
- アイコン表示の制御（テキスト併用 / アイコン単体）

## 3. 責務外
本コンポーネントは、以下を責務としない。

- タスク完了状態などの業務判定
- 業務ルールに基づく表示文言の決定
- 特定画面固有の状態管理

業務状態の判定は呼び出し元または業務専用コンポーネントで行う。

---

## 4. Props

| Props | 型 | 必須 | 説明 |
| --- | --- | :---: | --- |
| children | ReactNode |  | ボタン内に表示する内容（テキスト） |
| icon | ReactNode |  | ボタン内に表示するアイコン |
| iconOnly | boolean |  | アイコンのみで表示する場合に指定する |
| variant | `"primary" \| "secondary" \| "danger"` |  | ボタンの役割に応じた見た目を指定する |
| size | `"sm" \| "md"` |  | ボタンの表示サイズを指定する |
| disabled | boolean |  | 非活性状態を指定する |
| onClick | `() => void` |  | クリック時の処理を指定する |
| className | string |  | 標準スタイルに追加する個別調整用クラス |
| type | `"button" \| "submit" \| "reset"` |  | ボタン種別を指定する |

---

## 5. variant 定義

| variant | 用途 |
| --- | --- |
| primary | 画面内の主操作 |
| secondary | 補助操作 |
| danger | 破壊的操作 |

variant は業務種別ではなく、UI 上の役割として定義する。

---

## 6. size 定義

| size | 用途 |
| --- | --- |
| sm | カード内、一覧内などの小型ボタン |
| md | 通常サイズのボタン |

---

## 7. アイコン表示仕様

### 7.1 表示パターン
本コンポーネントは以下の表示をサポートする。

- テキストのみ
- アイコン + テキスト
- アイコンのみ（iconOnly）

### 7.2 iconOnly の扱い
- `iconOnly = true` の場合、アイコンのみを表示する
- children は表示対象外とする
- ボタンのサイズは正方形に近い形状とする（サイズ定義に依存）

例：
```tsx
<Button icon={<PlusIcon />} iconOnly />
```

### 7.3 設計方針
- iconOnly は「表示形式」を表すプロパティとする
- variant や size とは独立した軸として扱う
- variant に icon 用の値は追加しない

---

## 8. 使用方針

- 標準的な見た目は variant および size によって制御する
- 個別の軽微な調整が必要な場合のみ className を使用する
- 同一の個別調整が複数箇所で発生する場合は、共通化または variant 見直しを検討する

---

## 9. 設計上の補足

業務状態に応じた見た目の切り替えは呼び出し元で判定し、その結果を Props として Button に渡す。

例：

- 完了済みの場合は disabled = true
- 主操作として見せる場合は variant = "primary"
- 補助操作として見せる場合は variant = "secondary"

---

## 10. 備考

特定業務に依存するボタンが必要な場合は、本コンポーネントを直接拡張せず、業務専用のラッパーコンポーネントで吸収する。
