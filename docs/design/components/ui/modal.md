# Modal コンポーネント設計

## 1. 概要

`Modal` は、Taskuest v1 におけるオーバーレイ表示用の共通コンポーネントである。  
主に以下の用途で使用する。

- タスク作成モーダル
- タスク編集モーダル
- ユーザー情報表示モーダル
- 確認ダイアログの土台

本コンポーネントは、**モーダルの表示枠・閉じる操作・オーバーレイ表示といった共通UI責務に限定**し、  
モーダル内の業務ロジックや画面固有の内容は持たせない。

---

## 2. 目的

- モーダルUIの見た目と振る舞いを統一する
- task-list を含む複数画面で再利用できる表示基盤を作る
- 閉じる操作やオーバーレイの扱いを共通化する
- 過剰な機能追加を避け、v1 時点で保守しやすい設計にする

---

## 3. 配置場所

```txt
src/components/ui/Modal/
├─ Modal.tsx
├─ Modal.test.tsx
└─ index.ts
```

## 4. 責務

`Modal` が担当する責務は以下とする。

- モーダルの表示 / 非表示
- オーバーレイの表示
- モーダルコンテンツ領域の表示
- 閉じる操作の受け付け
- 共通スタイルの適用
- 必要に応じたタイトル表示

---

## 5. 担当しない責務

以下は `Modal` の責務に含めない。

- モーダル内の業務ロジック
- 入力値管理
- 保存 / 更新 / 削除処理
- 入力バリデーション
- API通信
- 確認ダイアログ固有の文言制御
- 複数モーダルの高度な管理
- 複雑なアニメーション制御

これらが必要になった場合は、画面側またはラッパーコンポーネント側で対応する。

## 6. 使用するHTML要素

- `div`（オーバーレイ）
- `div`（モーダル本体）
- `div`（ヘッダー / ボディ領域）
- `h2`（タイトル表示用）
- `button`（閉じる操作用）

## 7. Props 設計

| Props名 | 型 | 必須 | 説明 |
|---|---|---:|---|
| isOpen | `boolean` | ○ | 表示状態 |
| onClose | `() => void` | ○ | 閉じる操作時のイベント |
| children | `React.ReactNode` | ○ | モーダル内部に表示する内容 |
| title | `string` | △ | モーダルタイトル |
| showCloseButton | `boolean` | △ | 閉じるボタンを表示するかどうか。デフォルトは `true` |
| closeOnOverlayClick | `boolean` | △ | オーバーレイクリックで閉じるかどうか。デフォルトは `true` |
| className | `string` | △ | モーダル本体に追加調整が必要な場合の拡張用クラス |

### 7.1 設計判断

- `isOpen` によって表示 / 非表示を制御する
- `children` を使って内容を外から差し込む構成とする
- `title` は汎用モーダルとして最低限有用なため持たせる
- `showCloseButton` は閉じる導線を共通化するため持たせる
- `closeOnOverlayClick` は用途差分が出やすいため持たせる
- `size` や `variant` は現時点では持たせない  

---

## 8. 表示仕様

### 8.1 通常時
- `isOpen` が `true` の場合にオーバーレイとモーダル本体を表示する
- `children` に渡された内容をモーダル内に表示する

### 8.2 タイトル表示
- `title` が指定されている場合、モーダル上部にタイトルを表示する

### 8.3 閉じるボタン
- `showCloseButton={true}` の場合、閉じるボタンを表示する
- 閉じるボタン押下時は `onClose` を呼び出す

### 8.4 オーバーレイクリック
- `closeOnOverlayClick={true}` の場合、オーバーレイクリックで `onClose` を呼び出す
- モーダル本体内部のクリックでは閉じないようにする

### 8.5 非表示時
- `isOpen` が `false` の場合は描画しない、または非表示状態とする

---

## 9. アクセシビリティ

- モーダルであることが認識できるよう適切な属性付与を検討する
- タイトルがある場合、モーダルとタイトルを関連付ける
- キーボード操作で閉じるボタンへ到達できること
- 背景コンテンツとの差が視覚的に明確であること
- Esc キー対応は必要性が明確になった時点で追加検討する

## 12. 実装方針

- `Modal.tsx` を単体で実装する
- task-list のタスク作成 / 編集、およびユーザー情報表示で使用し、実利用を通じて不足を確認する
- 必要になるまでサイズ違いや高度なモーダル管理を持たせない
- 業務ロジックは `Modal` 本体ではなく、利用側コンポーネントで管理する

---

## 13. テスト観点

### 13.1 単体テスト対象
- `isOpen=true` のときにモーダルが表示されること
- `isOpen=false` のときに非表示であること
- `title` が表示されること
- `children` が表示されること
- `showCloseButton=true` のとき閉じるボタンが表示されること
- 閉じるボタン押下時に `onClose` が呼ばれること
- `closeOnOverlayClick=true` のときオーバーレイクリックで `onClose` が呼ばれること
- モーダル本体クリックでは `onClose` が呼ばれないこと

### 13.2 テストしないこと
- モーダル内フォームの業務ロジック
- API通信結果による表示制御
- 複数モーダルの管理
- 高度なアニメーション挙動
