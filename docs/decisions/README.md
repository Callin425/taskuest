# ADR (Architecture Decision Records)

## 目的
Taskuest における重要な設計判断と、その理由を記録するためのドキュメント。

## 対象
以下のような、後から見返したときに理由説明が必要になる判断を記録する。
- v1 / v2 のスコープ方針
- DB設計の方針
- 認証やアクセス制御の方針
- 画面構成やレイアウトの基本方針
- ドキュメント管理方針

## 記録しないもの
以下のような軽微な変更はADRではなく、詳細設計・コミット・PR説明で管理する。
- 文言修正
- 軽微なUI調整
- バグ修正
- 単一画面だけの細かい実装判断

## 命名ルール
- `0001-xxxx.md` のように連番で管理する
- 連番は作成順とする

## 運用ルール
- 重要な設計判断が発生したタイミングで追加する
- 既存方針を変更する場合は、必要に応じて新しいADRを追加する
- ADRは会話ログではなく、最終的な判断内容を簡潔に記録する
