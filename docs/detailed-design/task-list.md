# タスク一覧画面 詳細設計
## 1.画面概要
### 画面名
- タスク一覧画面
### 画面ID
- task-list
### 目的
- ログインユーザーが自身のタスクを一覧で確認し、追加・編集・完了・アーカイブを行うための画面。
### 利用レイアウト
- 認証後共通レイアウト
### 利用者
- ログインユーザー
### 関連機能
- タスク一覧表示
- タスク追加
- タスク編集
- タスク完了
- タスクアーカイブ

## 2. 画面レイアウト

### 2.1 画面構成

本画面は以下の要素で構成される。

- 共通ヘッダー
- サイドメニュー
- 画面タイトル
- タスク追加ボタン
- タスク並び替えコンボボックス
- タスク一覧表示エリア
- タスクカード
- 空状態表示
- タスク作成・編集モーダル

---

### 2.2 各エリア詳細

#### 共通ヘッダー
- ユーザーサムネイルを右上に表示する
- サムネイル押下でユーザーメニューを表示する
- ユーザーメニュー内にユーザー名・レベル・ログアウトを表示する

#### サイドメニュー
- タスク一覧画面への遷移
- プロジェクト一覧画面への遷移
- 現在表示中の画面をハイライト表示する

#### 画面タイトル
- 「タスク一覧」を表示する

#### タスク追加ボタン
- 押下時にタスク作成モーダルを表示する

#### タスク並び替えコンボボックス
- 標準はタスクの優先度で表示する
- 押下時にコンボボックスを展開する
- 選択肢として以下を表示する
  - 優先度
  - 作成日時（降順）
  - 更新日時（降順）
- 選択した条件でタスク一覧を再ソートして表示する
- ソート条件は状態として保持する

#### タスク一覧表示エリア
- ユーザーのタスク一覧を表示する
- アーカイブされていないタスクのみ表示する
- タスクはカード形式で表示する

#### タスクカード
- タスク名
- 説明
- 優先度
- ステータス
- 期限
- プロジェクトIDが設定されている場合はプロジェクト名を表示する
- タスク名は編集導線として扱い、押下時に対象タスクを編集モードでモーダル表示する
  - タスク名はクリック可能な表示とし、通常テキストと区別できる見た目とする
- 完了ボタンを表示し、押下時に対象タスクのステータスを完了に更新する
- アーカイブボタンを表示し、押下時に対象タスクをアーカイブ状態に更新する

#### 空状態表示
- タスクが存在しない場合に表示する
- 「タスクがありません」等のメッセージを表示する
- タスク追加ボタンを表示する

#### タスク作成・編集モーダル
- タスクの作成および編集を行う
- タスク追加ボタン押下時にcreateモードで表示する
- タスク名押下時にeditモードで表示する
- createモードでは入力項目を初期値空で表示する
- editモードでは対象タスクの既存値を初期表示する
- 入力項目として以下を表示する
  - タスク名
  - 説明
  - 優先度
  - 期限
  - プロジェクト
- タスク名はテキスト入力欄で表示する
  - 必須項目とする
  - 最大100文字まで入力可能とする
- 説明はテキストエリアで表示する
  - 任意項目とする
  - 最大500文字まで入力可能とする
- 優先度はセレクトボックスで表示する
  - 必須項目とする
  - 以下の選択肢を表示する
    - low
    - medium
    - high
    - urgent
- 期限は日付入力欄で表示する
  - 任意項目とする
- プロジェクトはセレクトボックスで表示する
  - 任意項目とする
  - ユーザーが所属しているプロジェクトを選択肢として表示する
  - 未設定を選択可能とする
- 保存ボタンを表示する
  - createモードでは新規作成処理を行う
  - editモードでは更新処理を行う
- キャンセルボタンを表示する
  - 押下時にモーダルを閉じる
- 必須項目未入力または不正な入力値の場合はエラーメッセージを表示し、保存処理を行わない
